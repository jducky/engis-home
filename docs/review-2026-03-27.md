# EnGIS 홈페이지 구조·콘텐츠 검토

검토일: 2026-03-27
대상 파일: `index.html`, `ai-solutions.html`

---

## 전체 구조 평가

사이트는 구조적으로 잘 잡혀 있고, B2B 공공 분야 기업 사이트로서 필요한 섹션(Hero → 서비스 → AI 솔루션 → 사례 → 강점 → 문의)을 모두 갖추고 있다. 다만 **콘텐츠의 신뢰도**와 **방문자 전환**을 높이려면 아래 부분을 보완해야 한다.

---

## 삭제하거나 수정해야 할 부분

### 1. 초안 흔적 문구 제거 (우선순위 높음)

현재 사이트 안에 아직 시안 단계의 설명 문구가 그대로 남아 있다.

- **Services 섹션 소개 문단**: `"현재 홈페이지의 사업분야, 기업소개, 연혁에 공개된 내용을 기준으로 보면..."` → 방문자에게 보여줄 문장이 아님. 엔지스의 강점을 직접 서술하는 문장으로 교체 필요.
- **Featured Cases 섹션**: `"아래 항목은 현재 홈페이지 연혁에 명시된 사업명을 중심으로 재구성한 메인 소개용 사례입니다. 운영용 사이트에서는..."` → 이 문장 전체 삭제.
- **Contact 섹션**: `"연락처 페이지 기준 공개 정보는 대표 노재덕·정진우..."`, `"운영용 사이트로 가려면..."` → 전부 삭제. 실제 문의 유도 문장으로 교체.
- **케이스 카드 bullet**: `"이력 확인"`, `"보여주는 사례 축으로 제시할 수 있습니다"` 같은 메타 표현 → 직접 설명하는 문장으로 전환.

### 2. `#targets` 섹션 (Target Customers) 위치 조정

현재 AI 솔루션 섹션 바로 다음에 배치되어 있는데, 서비스 → 사례 → 타겟 → 강점 순이 더 자연스럽다. 또는 각 서비스 카드 안에 "적합 고객" 태그로 흡수시키는 방안도 있다.

## 보완해야 할 내용

### 3. Hero 지표 (`hero-metrics`)

현재 `2013 / 공공·환경 / DB·GIS·AI` 세 개가 있는데, 숫자 기반 신뢰 지표가 아니다.
→ 예: `수행 프로젝트 30+`, `공공기관 납품 10년+` 같은 구체적 수치로 교체하면 설득력이 높아진다. 실제 수치를 확인 후 반영 필요.

### 4. 케이스 카드 내용 구체화

현재 케이스 카드의 bullet이 성과/결과가 아닌 설명 수준에 머물고 있다.
- `"2013년 이후 관련 구축·개선·유지관리 이력 공개"` → `"2013년~2023년 10년간 지속 수행"`처럼 수행 연속성을 드러내는 표현이 더 강하다.
- 가능하면 각 케이스에 **발주처명, 수행 범위, 결과 한 줄**을 추가.

### 5. `project-list` (하단 연혁형 리스트) 강화 또는 제거

현재 4개 항목이 간단한 설명과 연도로만 나열돼 있어 케이스 카드와 중복된다.
- 선택지 A: 더 많은 프로젝트를 연혁 형태로 나열 (2013~현재 전체)
- 선택지 B: 케이스 카드 3개로 통합하고 이 리스트 삭제
- 현재처럼 4개만 있으면 어중간하다.

### 6. `strengths` 섹션 시각적 약점

텍스트만 있는 4개 항목인데, 아이콘이나 수치 한 줄이 없어 앞의 `target-cards`보다 밀도가 낮다. 각 강점에 숫자나 기간을 한 줄 추가하거나, `target-section`과 통합을 고려할 수 있다.

### 7. Footer 없음 → **완료 (2026-03-27)**

공공기관 대상 사이트에서 신뢰 요소로 중요한 주소·사업자번호·저작권 표시를 추가했다.

### 8. `ai-solutions.html` CTA 버튼

Hero의 두 번째 버튼이 `"메인으로 돌아가기"` 인데, 솔루션 페이지 진입 초반에 이탈을 유도하는 버튼이다.
→ `"사례 보기"` 또는 `"Knowledge 보기"` 같이 페이지 내 탐색으로 바꾸는 게 낫다.

## 추가하면 좋은 내용

### 9. 회사 소개 섹션 부재

현재 사이트에는 **About/회사소개 섹션이 없다.** 설립연도·임직원 규모·인증 현황 등을 간단히 한 줄로라도 넣으면 공공 발주 고객의 신뢰에 도움이 된다.

### 10. 수상·인증·특허 섹션

2017년 해양수산 빅데이터 최우수상이 `project-list` 안에 묻혀 있는데, trust bar나 별도 신뢰 요소 구간으로 올려서 시각적으로 부각시키는 것이 좋다.
