# EnGIS 홈페이지 현황 분석 및 개편 방향

분석 기준일: 2026-03-25
분석 대상: `http://en-gis.com` / `http://www.en-gis.com`

## 1. 현재 상태 요약

- 현재 사이트는 정적 단일 페이지 중심 구조다.
- `https://www.en-gis.com` 및 `https://en-gis.com`은 접속되지 않고, `http`만 정상 응답한다.
- 메인 HTML 응답 헤더의 `Last-Modified`는 `2022-09-13`으로 확인된다.
- 첫 화면은 슬라이드형 비주얼 위주이며, 회사의 핵심 가치 제안이 즉시 명확하게 드러나지 않는다.
- 콘텐츠는 "서비스 소개", "회사 소개", "파트너 로고", "연락처" 수준에 머물러 있어 최근 B2B 기업 사이트 기준으로 정보 밀도가 낮다.

## 2. 핵심 문제 진단

### 2.1 브랜드 메시지가 분산되어 있음

- 첫 화면 슬라이드가 `정보시스템`, `GIS DB`, `AI 분석`, `작물질병진단`, `서울 랜드마크`, `미세먼지` 등 서로 다른 사례를 병렬로 보여준다.
- 방문자 입장에서 "엔지스가 정확히 어떤 문제를 어떤 고객에게 해결해주는 회사인지"가 한 문장으로 정리되지 않는다.
- 공공/환경/GIS/SI 전문 회사인지, AI 서비스 회사인지, 실험 프로젝트 쇼케이스 중심인지 포지셔닝이 모호하다.

### 2.2 정보 구조가 구식임

- 메인 페이지는 사실상 `Hero -> 회사 소개 3단 -> 파트너 로고 -> 푸터` 구조다.
- 최근 트렌드의 B2B 사이트가 보통 갖는 `핵심 서비스`, `대표 구축 사례`, `산업군`, `성과 지표`, `신뢰 요소`, `문의 전환` 구간이 없다.
- 파트너 로고는 있으나, 어떤 프로젝트를 수행했고 어떤 결과를 냈는지 설명이 없다.

### 2.3 전환 설계가 약함

- 메인 화면에서 즉시 클릭할 수 있는 강한 CTA가 없다.
- "상담 요청", "포트폴리오 보기", "사업 문의", "제안서 요청" 같은 행동 유도 장치가 부족하다.
- B2B 회사 사이트로서 리드 수집 구조가 약하다.

### 2.4 기술 신뢰감이 화면에 충분히 드러나지 않음

- 소개 문구는 포괄적이지만, 실제 전문성을 증명하는 자료가 부족하다.
- 기술 스택, 수행 분야, 주요 프로젝트 유형, 데이터 처리 역량, 운영 경험, 인증/특허/수상/레퍼런스가 눈에 띄게 정리되어 있지 않다.
- 회사의 강점이 "문장"으로만 존재하고 "증거"로 연결되지 않는다.

### 2.5 디자인 언어가 2022년 이전 스타일에 머물러 있음

- 전체 화면 슬라이드와 고정 사이드 도트 메뉴는 현재 기준으로 다소 구식 인상을 준다.
- 시각 계층이 약하고, 텍스트와 이미지가 서로 경쟁한다.
- 컬러 사용, 여백, 타이포그래피, 카드/섹션 구조가 현대적인 B2B SaaS 또는 테크 기업 사이트에 비해 평면적이다.
- 모바일에서 일부 본문이 숨겨지며 정보 손실이 발생한다.

### 2.6 접근성/SEO/운영 품질 이슈

- `https`가 동작하지 않는 상태는 신뢰와 검색 노출 모두에 악영향을 준다.
- 메타 설명, 오픈그래프, 구조화 데이터 등 기본 SEO 요소가 부족하다.
- jQuery를 2회 로드하고 있으며, `navbar`와 `footer`를 AJAX로 불러와 크롤링 친화성이 떨어진다.
- `javascript:void(0)` 링크, 일반적인 대체 텍스트, 스크롤 기반 색상 제어 등 접근성 품질이 낮다.
- 모바일에서 본문을 `display: none` 처리하는 구간이 있어 핵심 설명이 사라진다.

## 3. 가장 시급한 개선 우선순위

### P0. 신뢰 회복

- `https` 정상화 및 `http -> https` 리다이렉트
- 기본 SEO 메타 태그 정비
- 회사명, 핵심 서비스, 문의 경로를 첫 화면에서 즉시 보이게 수정

### P1. 메시지 재정리

- 회사의 핵심 포지셔닝을 1문장으로 재정의
- 메인 화면 슬라이드를 줄이고, 대표 메시지 1개와 보조 근거 3개 체계로 정리
- 잡다한 사례 나열 대신 "공공/환경/GIS/AI 데이터 활용"이라는 축으로 묶기

### P2. 정보 구조 개편

- `서비스`
- `구축 사례`
- `산업/고객군`
- `회사 역량`
- `문의`

위 5개 축 중심으로 메인 페이지 재설계

### P3. 디자인 현대화

- 풀페이지 캐러셀 중심 구조 제거 또는 축소
- 카드형 레이아웃, 명확한 섹션 구분, 데이터 중심 비주얼 적용
- 타이포와 여백 체계를 재정비해 B2B 테크 기업 톤 강화

## 4. 추천하는 새 메인 페이지 구조

### 4.1 Hero

- 헤드라인 예시: `공공·환경 분야의 GIS 데이터와 시스템을 설계하고 운영합니다`
- 서브카피: 데이터 구축, 공간분석, 시각화, 운영지원까지 한 번에 설명
- CTA 1: `사업 문의`
- CTA 2: `구축 사례 보기`
- 우측 또는 배경: 지도/레이어/대시보드 기반 비주얼

### 4.2 Trust Bar

- 주요 고객 기관 로고
- 설립연도, 수행 프로젝트 수, 운영 시스템 수, 전문 분야 수치

### 4.3 Core Services

- 공간정보 DB 구축
- GIS 기반 시스템 개발
- 환경/공공 데이터 분석 및 시각화
- AI 활용 분석/예측 서비스

각 서비스는 "무엇을 하는지"보다 "고객이 얻는 결과" 중심으로 기술

### 4.4 Featured Cases

- 대표 사업 3~4건
- 기관명, 문제 상황, 수행 범위, 결과를 카드로 정리
- 현재 사이트의 개별 서비스 예시는 이 구간으로 흡수하는 편이 적절함

### 4.5 Why EnGIS

- 공공사업 이해도
- GIS 전문성
- 데이터 구축부터 운영지원까지 가능한 수행 범위
- 환경/해양 등 도메인 경험

### 4.6 Contact CTA

- 전화, 이메일, 문의 버튼
- 제안/협업 문의를 유도하는 짧은 문구

## 5. 디자인 방향 제안

### 방향 A. Public Tech Premium

- 신뢰감 있는 네이비/슬레이트/티얼 계열
- 지도 레이어, 점/선/그리드, 대시보드 모티프 활용
- 공공/엔터프라이즈 고객에게 안정감 전달

### 방향 B. Data Visualization First

- 밝은 배경 위에 지도 조각, 차트, 흐름선, 레이어 UI 적용
- "공간 데이터 기업"이라는 인상을 직관적으로 전달
- 기존 사이트보다 훨씬 현대적인 느낌을 줄 수 있음

현재 회사 성격상 A를 기본으로 하고, B의 시각 요소를 일부 섞는 방식이 가장 적절하다.

## 6. 화면/콘텐츠 레벨의 구체적 개선 포인트

- 슬라이드 수를 1~2개 이하로 줄이기
- 영어/한국어 카피 혼합 방식을 정리하기
- 본문 카피를 "회사 자랑"보다 "고객 문제 해결" 중심으로 재작성하기
- 파트너 로고만 두지 말고 대표 레퍼런스 설명을 붙이기
- 모바일에서 숨겨지는 본문 제거
- 연락처 페이지 외에 메인에서도 바로 문의 가능하게 만들기
- 구성 요소를 AJAX 삽입 방식 대신 정적 렌더링 또는 프레임워크 기반 컴포넌트 구조로 전환하기
- 이미지 alt, 키보드 포커스, 명도 대비, 버튼 역할 등 접근성 기준 반영

## 7. 결론

현재 사이트는 회사의 기본 신뢰 정보는 전달하지만, 2026년 기준으로는 다음 문제가 크다.

- 보안/접속 신뢰 문제(`https` 미동작)
- 메시지 포지셔닝 불명확
- 사례와 성과 중심의 설득 구조 부족
- 디자인/인터랙션의 노후화

개편의 핵심은 "예쁘게 바꾸는 것"보다 아래 3가지를 동시에 해결하는 것이다.

- 엔지스가 누구를 위해 무엇을 잘하는 회사인지 5초 안에 이해되게 만들기
- 실제 수행 역량을 사례와 수치로 증명하기
- 문의 전환이 일어나도록 CTA와 정보 구조를 다시 설계하기
