  * 👉 “AI 챗봇 체험하기”
  * 👉 “우리 기술 보기”

### 핵심 기능 3개

# 🟦 2.2 AI 데모 페이지 (⭐ 핵심)

## 목적

👉 “이 회사는 실제로 AI 시스템 만들 줄 안다” 증명

---

## 📌 섹션 1: 회사 안내 챗봇

### 기능

* 회사 소개
* 실적 설명
* 기술 설명
* 문의 유도

### UI

* Chat UI
* 추천 질문 버튼

예:

* “어떤 사업을 하나요?”
* “GIS 관련 실적 있어?”
* “ODA 경험 있나요?”

---

## 📌 섹션 2: 문서 기반 Q&A (RAG)

### 기능

* 문서 기반 질의응답
* 출처 표시

### 데이터

* 회사소개서
* 실적자료
* 제안서 일부

### UI

* 질문 입력
* 답변 + 출처 링크

---

## 📌 섹션 3: DB 조회 데모

### 기능

👉 자연어 → DB 조회

### 예시 질문

* “AI 프로젝트만 보여줘”
* “2024년 이후 사업”
* “공공기관 프로젝트”

### 결과

* 표 형태 출력
* 필터/정렬

---

## 📌 섹션 4: DB 수정 데모

### 기능

👉 자연어 → DB 변경

### 예

* “3번 프로젝트 상태를 완료로 변경”
* “공지 하나 등록해줘”

### 필수 흐름

1. 요청 해석
2. 변경 미리보기
3. 사용자 승인
4. 실행

---

## 📌 섹션 5: 관리자 AI 도우미 (로그인)

### 기능

* 공지 자동 작성
* 실적 설명 생성
* 문의 분석
* 문서 요약

| 모드     | 설명       |
| ------ | -------- |

---

# 🧠 3.2 챗봇 처리 흐름

```
입력 → intent 분석 → 처리 분기
```

---

## 📌 1) 일반 질문

→ RAG

```
질문 → 벡터검색 → 문서 추출 → LLM 응답
```

---

## 📌 2) DB 조회 질문

→ SQL 생성

```
질문 → intent=조회 → SQL 생성 → DB 조회 → 결과 반환
```

---

## 📌 3) DB 수정 요청

→ 안전 처리

```
질문 → intent=수정 → SQL 생성 → 미리보기 → 승인 → 실행
```
